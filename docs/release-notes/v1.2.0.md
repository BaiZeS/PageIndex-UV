# Release Notes — v1.2.0

> Frozen at commit **`3d09544`**. Tag: `v1.2.0`.

## Headline

**Web console: modular refactor + visual overhaul + bug-fix pass.**

The single-file HTML/CSS/JS console shipped at v1.1 (`3a929c0`) was unmaintainable as a single 1190-line blob. v1.2 splits it into a layered design system + composables + focused components, and fixes two real bugs that were hiding under cached assets.

---

## What's new

### 1. Web console — modular split (19 new files)

| Layer | Files | Purpose |
|---|---|---|
| Design tokens | `tokens.css` | Single source of truth: color, type, spacing, radius, shadow, motion |
| Base / Element Plus overrides | `base.css` | Typography primitives + EP light-theme overrides |
| Component styles | `components.css` | Component-specific styles |
| Composable state | `lib/{api,auth,format,upload,qa,config}.js` | Fetch / API key / formatters / upload state / QA state / config state |
| Vue 3 components | `components/{panel,topbar,tab-nav,docs-panel,qa-panel,config-panel,confidence-meter,upload-progress-list,current-config-card}.js` | 9 focused SFCs-via-strings |
| Bootstrap | `app.js` | `createApp` + component registration + mount only — no business logic |

Each `useXxx()` composable is **singleton memoised** (`global.__useXxxSingleton`) so multiple call sites in different components share the same reactive refs.

### 2. C+A design system

Hybrid of **editorial calm** (Aesop / MUJI — C-school) and **builder SaaS precision** (Linear / Vercel — A-school):

- **Warm cream paper background** `#F7F2E7`, soft radial gradient overlays, paper-soft shadows.
- **Fraunces display serif** for headings + brand wordmark, **Inter** for UI, **JetBrains Mono** for paths / IDs / sizes / status chips.
- **Single deep-ink-blue accent** `#2E3D6B`. No purple-pink gradients. No emoji-as-icon.
- **3×3 SVG brand mark** in the topbar — `topbar.js` renders a 3×3 grid of rects with corner/center filled and edges at mid-opacity, evoking "indexed nodes in a tree".
- **Status chips** are mono + uppercase + pill + semantic color (success / warn / danger / accent).
- **Three-step type hierarchy**: eyebrow (mono caps) → title (Fraunces 26px) → description (Inter 13px muted).

See [`docs/web-console-design-system.md`](../web-console-design-system.md) for the full token + component catalog.

### 3. Bug fixes

#### RC1 — `<topbar />` self-closing swallowed in in-DOM template
- **Symptom**: page below topbar rendered completely blank after the refactor.
- **Mechanism**: Vue 3's in-DOM template compiler only treats standard HTML void elements (`img`, `br`, `input`, etc.) as self-closing. `<topbar>` is unknown → HTML parser nested `<main>` inside `<topbar>` → Topbar's template has no `<slot />` → children dropped silently.
- **Fix**: explicit close tag `<topbar></topbar>`.
- **Defense**: root template moved into a JS string in `app.js` so Vue uses the full compiler (which supports self-closing custom components) for any future root-level change.

#### RC2 — data not refetched when API key saved
- **Symptom**: user saved a key, console stayed empty until manual page reload.
- **Mechanism**: `saveKey()` wrote to localStorage but never re-invoked `loadDocs` / `loadConfig`.
- **Fix**: `saveKey()` now calls `loadDocs` + `loadConfig` after a non-empty save (try/catch guarded for singleton-not-yet-created).

#### Polish — `ConfidenceMeter` returned bare arrow functions from `setup`
- **Symptom**: template had to call `ratio()`, `label()`, `cssClass()` on every render. Lost reactive caching.
- **Fix**: wrapped each in `Vue.computed()`, return refs, template uses `ratio` / `label` / `cssClass` directly.

#### Polish — mount-time 403 noise
- **Symptom**: on first visit (empty localStorage), two red `Invalid or missing API Key` toasts surfaced before the user could type a key.
- **Fix**: post-mount `loadDocs` / `loadConfig` now guarded by `if (global.apiKey())`.

### 4. Server — disable HTTP caching for `/static/*`
- New `NoCacheStaticMiddleware` sets `Cache-Control: no-store, must-revalidate` on every `/static/*` response.
- External CDNs (Vue, Element Plus, fonts) keep their normal caching policies.
- Eliminates the "I deployed a fix but the browser still shows the old code" footgun.

## Migration / upgrade notes

- **No backend changes**: API contracts unchanged. Existing `X-API-Key` keeps working.
- **No DB migration**: SQLite schema unchanged.
- **`.env` unchanged format**; if you've set `API_KEY`, you keep using it.
- **Web console UI looks very different** from v1.1 — light theme instead of dark, different layout rhythm, but the same 3 tabs (Documents / Q&A / Model Config) and the same 4 MCP tools.
- **Browser cache**: after upgrading, hard-reload once (`Cmd/Ctrl + Shift + R`) so the new `tokens.css` + `base.css` + `components.css` aren't loaded from the old dark-theme cache. The new middleware makes this self-correcting going forward.

---

## What's NOT changed

- **`pageindex_mutil/`** (core retrieval, Super-Tree router, parsers, db): untouched.
- **MCP tool schemas**: unchanged (still `search`, `list_documents`, `get_document`, `delete_document`).
- **REST API paths**: unchanged.
- **CLI** (`main.py`): untouched.
- **Design doc system** (`docs/design-docs/PageIndex/<feature>/{spec,tasks}.md`): the v1.1 `web-console/spec.md` still describes v1.1 scope; this release notes covers v1.2.
